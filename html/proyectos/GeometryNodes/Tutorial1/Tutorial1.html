Entendiendo
geometry nodes

Geometry nodes system for modifying the geometry of an object with node-based operations. It can be accessed by adding a Geometry Nodes Modifier. Manual de blender

Atención

En este mini tutorial no pretendo hacer un sistema de hierba fotorealista valido para todos los casos. Solo pretendo familiarizarte en la modificación de posiciones de los vertices.

Muestra práctica

Hoja

Para generar la información de una hoja seria tan sencillo como modelar un plano y a este añadirle un Geometry Nodes. Seria lo sencillo ...sencillo.
No vamos a ir por ese camino. Para este caso generaremos esta información desde los nodos.

1

  • Generamos (con shft + a o como prefieras) un objeto cualquiera, en mi caso un cubo.
  • Le añadimos un modificador Geometry Node
  • Eliminamos los vértices del objeto, no los utilizaremos por el momento

2

Creamos un plano subdividido como nos convenga, en este caso para doblarlo hacia adelante y hacia atras.
Lo ponemos en vertical para que parezca una hoja.
Para que las coordenadas de los vertices esten en valores entre 0 y positivo, elevamos la maya la mitad de su longitud. Estoy seguro de que puedes ver el porqué.

3

Para continuar, la forma de la hoja rectangular puede ser interesante para minecraft pero, en nuestro caso, vamos a darle una forma mas intuitiva.

Las coordenadas en vertical (Z) son valores entre el 0 y un valor que llamaremos Sy. Sy lo conocemos, se trata de la altura maxima de la hoja y es un valor que elejimos nosotros con la variable global "Size y".

Normalizaremos estos valores y nos dará un rango entre 0 y 1. Ahora podemos decidir que si el valor es 0, la posición de los vertices en "x" sea 0, y si es 1, que se quede igual (traducido como una multiplicación). Obtenemos las coordenadas x de todos los vertices y las multiplicamos por el nuevo array de valores de 0 a 1 controlado por el ColorRamp.
Con este mismo metodo podemos desplazar los vertices hacia la "Y" en positivo.

Por el momento la coordenada "Y" es = a 0. Sumamos el valor de las coordenadas "Z" y obtenemos una recta.
Para la curva, al igual que antes, empleamos un ColorRamp.

4

Para terminar, la distribución de puntos se lleva a cabo con el siguiente grupo de nodos. Primero necesitamos geometria donde poner la hierba y luego distribuir puntos (densidad).
Para que no parezcan un ejercito de hojas, las rotamos aleatoriamente en la coordenada "Z".
Y tenemos nuestr oresultado final.
Para un sistema un poco más elaborado, podeis descargar el archivo con el arbol de nodos. Propongo que lo mireis por encima e intenteis entender las partes nuevas que se han añadido.
